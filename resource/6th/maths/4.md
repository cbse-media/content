## Introduction

We live in an age of information. If you ask your classmates about their favourite colours, or measure the weight of each student, you are collecting **Data**.

<Callout variant="tip">
**Definition:** Any collection of facts, numbers, measures, observations, or other descriptions of things that convey information about those things is called **data**.
</Callout>

In this chapter, we will explore how to collect data, organize it efficiently using tally marks, and present it visually using pictographs and bar graphs to make interpretations and inferences.

## Chapter Overview

This chapter covers the following key topics:

1.  **Collecting and Organising Data**: How to record data systematically using tables and tally marks.
2.  **Pictographs**: Representing data using symbols and pictures.
3.  **Bar Graphs**: Using bars of uniform width to represent data frequencies.
4.  **Artistic Considerations**: How the orientation and design of graphs can affect readability.

## Key Formulas & Concepts

| Term | Definition |
| :--- | :--- |
| **Data** | A collection of facts, numbers, or observations. |
| **Frequency** | The number of times a particular entry or value occurs. |
| **Tally Marks** | A method of counting frequencies using groups of 5 strokes (4 vertical, 1 diagonal). |
| **Scale** | The value represented by one unit length on a bar graph or one symbol in a pictograph (e.g., 1 unit = 10 students). |
| **Bar Graph** | A visual representation of data using bars of uniform width and varying heights. |
| **Pictograph** | A visual representation of data using pictures or symbols. |

Let's begin by learning how to collect and organize raw data.

## What is Data?

Data is any collection of facts—such as numbers, words, measurements, or observations—that gives us information. For example, a list of your friends' favourite games or the temperatures recorded over a week is data.

When data is first collected, it is often unorganized. To make sense of it, we need to organize it.

## Organizing Data

Imagine a teacher wants to know the favourite sweets of students in a class. If he just writes down names as students shout them out, he gets a messy list:
*Jalebi, Gulab Jamun, Rasgulla, Jalebi, Jalebi, Barfi...*

To organize this, we use a **Table**. We list the items (Sweets) in one column and the count (Number of Students) in another.

### Tally Marks

Counting large amounts of data can be tricky. To avoid mistakes, we use **Tally Marks**.

<Callout variant="tip">
**Tally Mark System:**
- We draw a vertical line `|` for each count.
- When the count reaches 5, we cross the previous four lines diagonally.
- This creates a group of 5: `||||` crossed becomes a bundle.
</Callout>

This makes it easy to count in multiples of 5 (5, 10, 15...).

## What is a Pictograph?

A **Pictograph** represents data through pictures of objects. It helps answer questions about data with just a quick glance. Instead of looking at numbers, you look at symbols.

### Key Components

1.  **Symbol**: A picture that represents the item (e.g., a smiley face for a student, a bulb for electricity).
2.  **Scale/Key**: This is crucial. It tells you what **one symbol** represents.
    *   Example: $1 \text{ Smiley} = 1 \text{ Student}$
    *   Example: $1 \text{ Smiley} = 10 \text{ Students}$

If the scale is large (e.g., 1 symbol = 10 items), we might use partial pictures to represent smaller numbers.
*   Half a symbol $\approx 5$ items.

## What is a Bar Graph?

A **Bar Graph** allows us to quickly understand and interpret information by using bars of uniform width.

<Callout variant="tip">
**Key Features:**
1.  **Uniform Width:** All bars must be the same width.
2.  **Equal Spacing:** The gap between each bar must be the same.
3.  **Height/Length:** The height of the bar represents the frequency (number) of the data.
</Callout>

## Drawing a Bar Graph

1.  **Draw Axes:** Draw a horizontal line (x-axis) and a vertical line (y-axis).
2.  **Choose a Scale:** Determine how many units 1 cm represents. (e.g., 1 unit = 100 vehicles).
3.  **Mark Categories:** Mark the items (e.g., time intervals) on the horizontal axis with equal gaps.
4.  **Draw Bars:** Draw vertical bars for each item. The height is determined by dividing the frequency by the scale.
    *   Example: If frequency is 600 and scale is 1 unit = 100, height = $600 / 100 = 6$ units.

## Vertical vs. Horizontal Bars

Does it matter if the bars go up or sideways?
While the data remains the same, the presentation can make it more intuitive.

*   **Vertical Bars (Column Graphs):** Best for heights or amounts measured upwards from the ground (e.g., height of mountains, number of students).
*   **Horizontal Bars:** Best for lengths parallel to the ground or distances (e.g., length of rivers, running distances).

### Example: Tallest Mountains

Since mountains grow upwards, a vertical bar graph is visually suggestive of the subject matter.

### Example: Longest Rivers

Rivers flow across the land. A horizontal bar graph mimics the flow of a river better than a vertical column.

## Infographics

When graphs are beautified with artistic imagery, they are called **Infographics**.
Instead of plain rectangular bars, an artist might use:
*   Triangles to represent mountains.
*   Pencils to represent school supplies.

<Callout variant="warning">
**Caution:** Fancy graphics can sometimes be misleading! If you use a triangle to represent a mountain, a taller triangle is also *wider* at the base. This might make the data look much larger than it actually is. Always check the scale and the axis values, not just the size of the picture.
</Callout>

## Example 2: Interpreting Tally Marks

**Problem:**
A teacher marked attendance using tally marks. For a specific day, the mark was: `||||` `||||` `||`.
How many students were present?

**Solution:**
1.  Identify complete bundles: There are 2 bundles of `||||` (crossed).
    *   $2 \times 5 = 10$.
2.  Identify loose lines: There are 2 loose lines `||`.
    *   $10 + 2 = 12$.
3.  **Answer:** 12 students.
